\begin{abstract}
\noindent 本次大作业基于课程提供的 C++ 光线追踪框架，完成了一个 Whitted 风格的光线追踪渲染器。
具体实现了四个核心函数：\texttt{Renderer::Render}（主光线生成）、\texttt{Triangle::getIntersection}
（光线–三角形求交，Möller–Trumbore 算法）、\texttt{Bounds3::IntersectP}（光线–包围盒求交，slab 测试）
与 \texttt{BVHAccel::getIntersection}（BVH 层次遍历加速求交）；并额外完成了 \texttt{Scene::castRay\_noBVH}
中的完整着色模型（加分项）。在此基础上，本文对渲染器做了系统的\textbf{性能与质量优化}：BVH 遍历改为
\textbf{有序遍历 + 最近距离剪枝}、构建采用 \textbf{SAH（表面积启发式）}、阴影光线改为\textbf{带光源距离
上界的 any-hit 提前退出}、主渲染循环用 \textbf{OpenMP 多线程}并行，并补充了\textbf{逐三角形抗锯齿超采样}
与真正生效的 \textbf{Phong 高光}；同时修正了若干正确性问题（阴影距离校验、阴影光线背面剔除导致的漏光）。
在 $1280\times960$、含 \textbf{4968} 面的 Stanford Bunny 上，优化后 BVH 单线程仅需 \textbf{0.24\,s}、32 线程
约 \textbf{40\,ms}；并通过自动化实验在同一 32 线程下对比四个不同面数模型在 BVH 与逐三角形暴力下的渲染耗时，
BVH 加速比最高达 \textbf{6588} 倍，且渲染耗时几乎不随面数增长。本文逐一说明各函数的算法原理、关键代码、
所做优化与实验结果，并对全部结论给出可复现的真实测量。
\end{abstract}

\section{概述}

光线追踪通过\textbf{逆向}追踪从相机出发、穿过每个像素射入场景的光线，求出光线与场景的最近交点并在该点着色，
从而生成图像。当场景中三角形数量较大时，"对每条光线遍历所有三角形" 的暴力做法效率极低；为此需要使用
\textbf{层次包围盒（Bounding Volume Hierarchy, BVH）}对求交过程进行加速：先用廉价的光线–包围盒测试快速
排除大片不可能相交的空间，再递归深入到真正可能相交的少数三角形。整体渲染流程如图~\ref{fig:pipeline} 所示。

\begin{figure}[H]
  \centering
  \includegraphics[width=\textwidth]{figures/pipeline.png}
  \caption{光线追踪渲染流程：相机生成主光线 $\rightarrow$ BVH 遍历剪枝 $\rightarrow$ Möller–Trumbore
  光线–三角形求交 $\rightarrow$ Phong 着色（含阴影测试）$\rightarrow$ 写入帧缓冲并输出 PPM 图像。}
  \label{fig:pipeline}
\end{figure}

本框架需要在以下文件中补全代码：\texttt{Renderer.cpp}、\texttt{Triangle.hpp}、\texttt{Bounds3.hpp}、
\texttt{BVH.cpp}，并可选地在 \texttt{Scene.cpp} 中补全着色模型。下面分别说明。

\section{各函数的实现}

\subsection{主光线生成：\texttt{Renderer::Render}}

该函数遍历图像的每个像素 $(i,j)$，构造一条从相机出发、穿过该像素中心的光线。设视场角为 \texttt{fov}，
则 $\text{scale}=\tan(\text{fov}/2)$，图像宽高比 $\text{aspect}=\text{width}/\text{height}$。
将像素坐标映射到位于 $z=-1$ 平面上的相机空间坐标：
\begin{align}
x &= \bigl(2\cdot\tfrac{i+0.5}{\text{width}}-1\bigr)\cdot\text{aspect}\cdot\text{scale},\\
y &= \bigl(1-2\cdot\tfrac{j+0.5}{\text{height}}\bigr)\cdot\text{scale}.
\end{align}
其中 $+0.5$ 使光线穿过像素中心，$y$ 方向取负是因为图像行号自上而下增大而相机 $y$ 轴向上。
随后归一化方向并投射光线：

\begin{lstlisting}
float x = (2.0f * (i + 0.5f) / scene.width  - 1.0f) * imageAspectRatio * scale;
float y = (1.0f - 2.0f * (j + 0.5f) / scene.height) * scale;
Vector3f dir = normalize(Vector3f(x, y, -1));
Ray ray(eye_pos, dir);
framebuffer[m++] = check_mode ? scene.castRay_noBVH(ray, 0) : scene.castRay(ray, 0);
\end{lstlisting}

最终把浮点颜色 \texttt{clamp} 到 $[0,1]$ 并按 P6 格式写出 \texttt{binary.ppm}。

\subsection{光线–三角形求交：\texttt{Triangle::getIntersection}（Möller–Trumbore）}

光线 $\mathbf{O}+t\mathbf{D}$ 与三角形 $(\mathbf{P}_0,\mathbf{P}_1,\mathbf{P}_2)$ 的交点可用重心坐标 $(u,v)$ 表示：
\begin{equation}
\mathbf{O}+t\mathbf{D}=(1-u-v)\mathbf{P}_0+u\mathbf{P}_1+v\mathbf{P}_2.
\end{equation}
令 $\mathbf{E}_1=\mathbf{P}_1-\mathbf{P}_0,\ \mathbf{E}_2=\mathbf{P}_2-\mathbf{P}_0,\ \mathbf{T}=\mathbf{O}-\mathbf{P}_0$，
由 Cramer 法则（Möller–Trumbore）可得
\begin{equation}
\begin{bmatrix}t\\u\\v\end{bmatrix}
=\frac{1}{\mathbf{E}_1\cdot(\mathbf{D}\times\mathbf{E}_2)}
\begin{bmatrix}
\mathbf{E}_2\cdot(\mathbf{T}\times\mathbf{E}_1)\\
\mathbf{T}\cdot(\mathbf{D}\times\mathbf{E}_2)\\
\mathbf{D}\cdot(\mathbf{T}\times\mathbf{E}_1)
\end{bmatrix}.
\end{equation}
有效交点必须满足 $u\ge0,\ v\ge0,\ u+v\le1$（落在三角形内）且 $t\ge0$（位于光线前方）。实现上：

\begin{lstlisting}
Vector3f pvec = crossProduct(ray.direction, e2);
float det = dotProduct(e1, pvec);
if (fabs(det) < EPSILON) return inter;          // 光线与三角形平行

float invDet = 1.0f / det;
Vector3f tvec = ray.origin - v0;
float u = dotProduct(tvec, pvec) * invDet;
if (u < 0 || u > 1) return inter;
Vector3f qvec = crossProduct(tvec, e1);
float v = dotProduct(ray.direction, qvec) * invDet;
if (v < 0 || u + v > 1) return inter;
float t = dotProduct(e2, qvec) * invDet;
if (t < 0) return inter;                         // 交点在光线起点之后

inter.happened = true;  inter.coords = ray(t);
inter.normal = normal;  inter.distance = t;
inter.obj = this;       inter.m = m;
\end{lstlisting}

\noindent\textbf{实现要点}：相比框架自带的辅助函数，这里在函数内部直接实现了 Möller–Trumbore，
并补充了两处健壮性判断——平行判定使用 $|det|<\varepsilon$（而非严格等于 0），以及 $t\ge0$
剔除位于相机背后的交点，避免错误交点污染 BVH 的最近距离比较。
此处\textbf{不做背面剔除}（早期版本曾剔除背面）：对闭合网格而言最近可见面仍是正面、渲染结果不变，
但保留双面求交可让\textbf{阴影光线}正确命中朝向光源的遮挡面，避免单面遮挡物漏光（详见性能优化一节）。

\subsection{光线–包围盒求交：\texttt{Bounds3::IntersectP}（slab 测试）}

轴对齐包围盒（AABB）可视为三对相互垂直的平行平面（slab）。光线进入第 $k$ 个 slab 的参数为
$t^{\min}_k=(p^{\min}_k-O_k)/D_k$，离开为 $t^{\max}_k=(p^{\max}_k-O_k)/D_k$。光线进入整个盒子的时刻为
各轴进入时刻的最大值，离开盒子的时刻为各轴离开时刻的最小值：
\begin{equation}
t_{\text{enter}}=\max_k t^{\min}_k,\qquad
t_{\text{exit}}=\min_k t^{\max}_k.
\end{equation}
当且仅当 $t_{\text{enter}}\le t_{\text{exit}}$ 且 $t_{\text{exit}}\ge0$ 时光线与盒子相交（用 $\le/\ge$ 比严格
不等式更稳健，兼容轴对齐相切的退化情形）。为避免除法，传入预先算好的 \texttt{invDir}$=1/\mathbf{D}$。这里采用
\textbf{与符号无关的逐分量 min/max} 写法（而非按方向符号 \texttt{swap}）：当某轴方向分量恰为 $0$、即
\texttt{invDir} 为 $\pm\infty$ 时，该轴自然得到 $(-\infty,+\infty)$ 的无约束区间，避免 \texttt{swap} 写法在该
退化情形下误判未命中。为支持后文的有序遍历与剪枝，把该测试写为返回\textbf{进入时刻 $t_{\text{enter}}$}
（未命中返回 $+\infty$）的版本 \texttt{IntersectT}：

\begin{lstlisting}
Vector3f t1 = (pMin - ray.origin) * invDir;
Vector3f t2 = (pMax - ray.origin) * invDir;
float t_enter = max( min(t1.x,t2.x), min(t1.y,t2.y), min(t1.z,t2.z) );
float t_exit  = min( max(t1.x,t2.x), max(t1.y,t2.y), max(t1.z,t2.z) );
if (t_exit < 0 || t_enter > t_exit) return INF;  // 未命中
return t_enter;                                  // 命中：返回进入时刻（可<0，起点在盒内）
\end{lstlisting}

  启发式选择划分位置，退化时回退质心中位数），叶节点对应单个三角形；

\subsection{BVH 遍历求交：\texttt{BVHAccel::getIntersection}（有序遍历 + 最近距离剪枝）}

建立 BVH 后，求交转化为对二叉树的遍历。最朴素的写法是无条件递归左右子树再取更近者；本文在保证结果正确的
前提下做了两点优化：（1）\textbf{有序遍历}——先进入\textbf{包围盒入口更近}的子树；（2）\textbf{最近距离剪枝}
——用当前已找到的最近交点距离 \texttt{best.distance} 作上界，凡是\textbf{盒入口已比它更远}的子树直接跳过。
\texttt{invDir} 与 \texttt{dirIsNeg} 在每条光线进入时只计算一次并向下传递（不再每个节点重算）：

\begin{lstlisting}
float tBox = node->bounds.IntersectT(ray, invDir, dirIsNeg);
if (!(tBox < best.distance)) return;             // 未命中或盒比当前最近交点更远 -> 剪枝
if (node->left == nullptr && node->right == nullptr) {           // 叶节点
    Intersection it = node->object->getIntersection(ray);
    if (it.happened && it.distance < best.distance) best = it;
    return;
}
float tL = node->left->bounds.IntersectT(ray, invDir, dirIsNeg);
float tR = node->right->bounds.IntersectT(ray, invDir, dirIsNeg);
auto first  = (tL <= tR) ? node->left  : node->right;            // 先近后远
auto second = (tL <= tR) ? node->right : node->left;
float tSecond = (tL <= tR) ? tR : tL;                            // 远侧子树的盒入口时刻
getIntersection(first,  ray, invDir, dirIsNeg, best);
if (tSecond < best.distance)                                     // 远侧仅在更近时才下探
    getIntersection(second, ray, invDir, dirIsNeg, best);
\end{lstlisting}

\noindent 由于 \texttt{Intersection} 的默认 \texttt{distance} 为 \texttt{double} 最大值，未命中的子树自然
不会赢得最近距离比较。相比朴素遍历，"先近后远 + 距离剪枝"可在找到较近交点后跳过大量更远的子树，
单条光线实际访问的节点数明显减少（实测见性能优化一节）。此外，针对\textbf{阴影光线}另写了 any-hit 版本
\texttt{IntersectP(ray, tMax)}：只需判断"光源与交点之间是否存在遮挡"，因此命中第一个 $t<t_{\max}$ 的
三角形即可立即返回、无需求最近交点。

\subsection{额外内容：\texttt{castRay\_noBVH} 的完整着色}

框架在检测模式（\texttt{./RayTracing check}）下不会调用 \texttt{buildBVH()}，故 \texttt{scene.bvh} 为空，
原 \texttt{castRay\_noBVH} 仅返回纯白以验证求交是否正确。本部分按文档"额外内容"要求，参照上方的
\texttt{castRay} 补全了完整的 Whitted 着色模型。为不依赖场景级 BVH，新增了暴力最近交点函数：

\begin{lstlisting}
Intersection Scene::intersect_noBVH(const Ray &ray) const {
    Intersection nearest;                       // 默认 distance = DBL_MAX
    for (uint32_t k = 0; k < objects.size(); ++k) {
        Intersection it = objects[k]->getIntersectionNoBVH(ray);   // 真.逐三角形暴力
        if (it.happened && it.distance < nearest.distance) nearest = it;
    }
    return nearest;
}
\end{lstlisting}

\noindent 这里的 \texttt{getIntersectionNoBVH} 对网格物体\textbf{直接遍历其全部三角形}、绕过
\texttt{MeshTriangle} 内部 BVH，从而提供真正不含任何层次结构的暴力求交基线（用于后文的加速实验）。
\texttt{castRay\_noBVH} 在此基础上完成与 \texttt{castRay} 一致的三分支着色：
（1）反射 + 折射材质用 Fresnel 方程混合反/折射颜色；（2）镜面反射材质追投反射光线；
（3）漫反射/光泽材质使用 Phong 模型（漫反射 + 高光），并对每个光源投射\textbf{阴影光线}判断遮挡：
\begin{equation}
L=\sum_{\text{lights}} (1-\text{inShadow})\,I\,(\mathbf{N}\cdot\mathbf{l})\,k_d\,c_d
   + I\,\max(0,-\mathbf{r}\cdot\mathbf{d})^{p}\,k_s.
\end{equation}
本场景的网格材质设为 $k_d=0.6,\ k_s=0.3,\ p=64$，因此 Phong 的\textbf{镜面高光项真正生效}（早期版本
$k_s=0$ 实际只有漫反射，无高光）；阴影测试使用带光源距离上界的 any-hit 遍历 \texttt{intersectP}。
反射/折射两分支的代码已实现并正确，但本次交付场景只放置一个漫反射/光泽材质的网格，故实际只走 Phong 分支，
反射/折射分支不被触发（如需展示可另加镜面/玻璃球）。检测模式与 BVH 模式最终得到\textbf{逐像素完全一致}的着色图像。

\section{性能与质量优化}

在四个核心函数正确工作的基础上，本文对渲染器做了一轮系统优化，分为正确性、性能与画质三类。

\paragraph{正确性修正}（1）\textbf{阴影距离校验}：原阴影测试仅判断阴影光线是否命中任意物体，未校验遮挡物
是否比光源更近，导致\textbf{光源之后}的几何也会被误判为遮挡；现以光源距离 $\lVert \text{light}-\text{hit}\rVert$
作为上界。（2）\textbf{阴影光线不背面剔除}：求交统一去掉背面剔除，使朝向光源的单面遮挡物也能正确挡光、消除漏光。
（3）\textbf{反射光线自交偏移}：修正 \texttt{REFLECTION} 分支沿表面法线偏移的符号，使其与折射分支约定一致、不再偏入表面内侧。

\paragraph{求交加速}（1）\textbf{有序遍历 + 最近距离剪枝}：BVH 遍历改为先进入包围盒入口更近的子树，并用当前
最近交点距离剪掉入口更远的子树（见上一节）；\texttt{invDir}/\texttt{dirIsNeg} 每条光线只算一次。
（2）\textbf{SAH 构建}：BVH 构建从中位数划分改为表面积启发式（SAH）分桶，按 $\text{SA}(L)\,n_L+\text{SA}(R)\,n_R$
最小选择划分位置，得到更平衡、遍历代价更低的树。（3）\textbf{阴影 any-hit 提前退出}：阴影查询用 \texttt{IntersectP}，
命中第一个 $t<t_{\max}$ 的三角形即返回。（4）顺手删除了构建期一段从未使用的全量包围盒求并（死代码）。

\paragraph{并行}主渲染循环用 \textbf{OpenMP} 在像素行上并行（\texttt{\#pragma omp parallel for}）。
由于每个像素独立、着色函数为 \texttt{const} 且无共享可变状态，唯一改动是把原先依赖跨迭代自增计数器
\texttt{framebuffer[m++]} 改为与迭代无关的索引 \texttt{framebuffer[j*width+i]}，从而消除伪依赖。

\paragraph{画质}（1）\textbf{抗锯齿}：支持每像素 $N\times N$ 规则超采样（命令行 \texttt{spp=N}），对物体轮廓与
阴影边缘做平均，显著减轻锯齿（计时实验仍用 \texttt{spp=1} 以与原方法学一致，展示图用 \texttt{spp=2}）。
（2）\textbf{真实高光}：网格材质 $k_s$ 由 $0$ 改为 $0.3$、$p$ 由 $0$ 改为 $64$，使 Phong 高光真正出现在渲染结果中。

\paragraph{工程}\texttt{CMakeLists.txt} 未显式指定时默认 \texttt{Release}（\texttt{-O3}）并链接 OpenMP，
避免误用未优化构建测量性能。所有以上改动都保证 BVH 模式与检测模式输出\textbf{逐像素一致}（见下节验证）。

\section{总结}

本次作业完整实现了基于 BVH 加速的 Whitted 风格光线追踪器：从主光线生成、Möller–Trumbore
光线–三角形求交、AABB slab 包围盒测试，到 BVH 遍历，并额外补全了非 BVH 路径下的完整着色模型；
在此之上做了系统优化——有序遍历 + 最近距离剪枝、SAH 构建、阴影 any-hit、OpenMP 多线程、抗锯齿与真实高光，
并修正了阴影距离校验等正确性问题。优化后四千余面的 Bunny 场景 BVH 单线程约 $0.24\,\mathrm{s}$、32 线程约
$40\,\mathrm{ms}$。进一步的跨模型对照实验表明，在同一 32 线程下，相比逐三角形暴力求交，BVH 在 $4968$ 至
$34.6$ 万面的模型上取得 $36$ 至 $6588$ 倍加速，且渲染耗时几乎不随面数增长（单线程 $0.24\to0.32\,\mathrm{s}$，
面数 $\times70$ 而时间仅 $\times1.3$），凸显了层次包围盒在复杂场景下的必要性。通过本次实践，加深了对
光线生成、解析法求交、空间数据结构加速、并行化以及局部光照（Phong + 阴影）等图形学核心概念的理解。
